# Requirements: Logwell Go SDK CI

**Defined:** 2026-02-26
**Core Value:** Every change to the Go SDK is automatically tested, linted, and validated before merge — no broken SDK releases reach users.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Workflow Triggers

- [x] **TRIG-01**: Workflow triggers on push to main affecting `sdks/go/**` or `.github/workflows/sdk-go.yml`
- [x] **TRIG-02**: Workflow triggers on pull request affecting `sdks/go/**` or `.github/workflows/sdk-go.yml`
- [x] **TRIG-03**: Workflow supports manual dispatch via `workflow_dispatch`
- [x] **TRIG-04**: Concurrent runs on same branch cancel in-progress with `cancel-in-progress: true`

### Linting

- [x] **LINT-01**: `golangci-lint` v2 runs via `golangci/golangci-lint-action@v9` with pinned version
- [x] **LINT-02**: `.golangci.yml` config file exists at `sdks/go/` with `version: "2"` top-level key
- [x] **LINT-03**: Lint job uses explicit `working-directory: sdks/go` in action `with:` block (not job defaults)
- [x] **LINT-04**: `go vet ./...` runs as part of lint or test pipeline

### Testing

- [x] **TEST-01**: `go test -race -count=1 -v ./...` runs on every trigger
- [x] **TEST-02**: Tests run in a matrix across Go 1.25.x and 1.26.x (current supported releases)
- [x] **TEST-03**: `GOTOOLCHAIN=local` is set to prevent auto-upgrade during matrix runs
- [x] **TEST-04**: Test matrix uses `fail-fast: false` to report all failures

### Coverage

- [x] **COV-01**: Coverage profile generated with `-covermode=atomic -coverprofile=coverage.out`
- [x] **COV-02**: Coverage summary written to `$GITHUB_STEP_SUMMARY` via `go tool cover`
- [x] **COV-03**: Coverage artifact uploaded via `actions/upload-artifact`

### Infrastructure

- [x] **INFR-01**: All `run:` steps execute from `sdks/go/` via `defaults.run.working-directory`
- [x] **INFR-02**: `actions/setup-go@v6` with `cache-dependency-path: sdks/go/go.sum` for module cache
- [x] **INFR-03**: `ci-success` gate job aggregates all jobs with `if: always()` for stable branch protection
- [x] **INFR-04**: All jobs have `timeout-minutes: 10` to prevent runaway runners

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Security

- **SEC-01**: `govulncheck` runs via `golang/govulncheck-action` on PRs and weekly schedule
- **SEC-02**: `govulncheck` results reported to GitHub Security tab via SARIF

### Quality Gates

- **QUAL-01**: `go mod tidy` drift check fails CI if `go.mod` or `go.sum` are stale
- **QUAL-02**: Coverage threshold enforcement at measured baseline (set after Phase 1 runs)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Cross-platform matrix (Windows/macOS) | Pure Go stdlib library, no platform-specific behavior; triples CI cost |
| Release automation (goreleaser) | Go libraries publish to pkg.go.dev via git tag, no tooling needed |
| Codecov/Coveralls integration | `$GITHUB_STEP_SUMMARY` sufficient; avoids external tokens and accounts |
| Integration tests against live Logwell | Requires infrastructure; unit tests with httptest are sufficient |
| `staticcheck` standalone | Already bundled in golangci-lint |
