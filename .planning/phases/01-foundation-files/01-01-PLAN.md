<task type="auto">
  <name>Task 1: Create golangci-lint v2 config and bump go.mod minimum version</name>
  <files>sdks/go/.golangci.yml, sdks/go/go.mod</files>
  <action>
1. Create `sdks/go/.golangci.yml` with the following exact content (from RESEARCH.md verified pattern):

```yaml
version: "2"

linters:
  default: none
  enable:
    - govet
    - errcheck
    - staticcheck
    - ineffassign
    - unused

formatters:
  enable:
    - gofmt
    - goimports
```

Critical: `version: "2"` MUST be the first substantive line. Formatters (`gofmt`, `goimports`) go under `formatters.enable`, NOT under `linters.enable` — this is a v2 breaking change from v1. Use `linters.default: none` to disable all defaults and explicitly enable only the five practical linters per FOUND-02.

2. Edit `sdks/go/go.mod` line 3: change `go 1.21` to `go 1.22`. Use the language version form (no patch suffix). The file should read:

```
module github.com/Divkix/Logwell/sdks/go

go 1.22
```

3. Run `go mod tidy` from `sdks/go/` to ensure go.sum stays in sync after the version bump (the SDK has no external deps so this should be a no-op, but prevents Phase 3 LINT-04 from failing on stale go.sum).

<task type="auto">
  <name>Task 2: Validate golangci-lint config parses without errors</name>
  <files>sdks/go/.golangci.yml</files>
  <action>
Run golangci-lint against the config to verify it parses correctly under v2 schema rules.

From the `sdks/go/` directory, run:
```bash
golangci-lint run --config .golangci.yml ./...
```

Expected outcomes:
- Exit code 0 = config valid AND no lint issues in current code
- Exit code 1 with lint issues = config valid, but existing code has lint violations (acceptable for Phase 1 — Phase 3 handles lint compliance)
- Error mentioning "unknown field" or "invalid configuration" = config schema is WRONG — fix immediately

The success criterion for Phase 1 is that the config **parses without schema errors**, not that the code is lint-clean. If golangci-lint exits with lint findings (not config errors), that is a PASS for this task.

If golangci-lint is not installed locally, install it:
```bash
# macOS
brew install golangci-lint
# or via go install
go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@latest
```

Also verify the go.mod version bump is correct:
```bash
cd sdks/go && go vet ./...
```
This ensures the Go toolchain accepts the `go 1.22` directive without complaints.
